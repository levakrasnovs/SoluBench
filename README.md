# SoluBench

**SoluBench** is a benchmark for evaluating large language models on solubility-related tasks of various complexity. It is built on top of [BigSolDB v2.0](https://www.nature.com/articles/s41597-025-05559-8) and [MixtureSolDB](https://www.nature.com/articles/s41597-026-07047-z) — two curated experimental solubility datasets.

> 📄 Paper: [Can LLMs Solve Solubility Tasks? The SoluBench Benchmark for Pure and Mixed Solvent Systems](https://chemrxiv.org/doi/abs/10.26434/chemrxiv.15000632)

---

## Tasks

| # | Name | Description | Input | Output | n | Random baseline |
|---|------|-------------|-------|--------|---|----------------|
| 1 | Pairwise solvent comparison | Given a compound (SMILES) and two solvents, select the one with higher solubility | SMILES + 2 solvents | A / B | 3699 | 50.0% |
| 2 | Best solvent selection | Given a compound and a list of solvents, select the one with highest solubility | SMILES + N solvents | letter | 878 | ~17.1% |
| 3 | Co-solvent effect prediction | Given a compound, a base solvent, and a co-solvent mixture, predict whether solubility increases or decreases | SMILES + mixture composition | A / B | 2229 | 50.0% |
| 4 | Pairwise compound comparison | Given a solvent and two compounds (SMILES), select the one with higher solubility | Solvent + 2 SMILES | A / B | 3000 | 50.0% |

## Installation

```bash
git clone https://github.com/levakrasnovs/SoluBench.git
cd SoluBench
pip install -r requirements.txt
```

Set your [OpenRouter](https://openrouter.ai) API key:

```bash
echo "OPENROUTER_API_KEY=sk-or-..." > .env
```

---

## Running a model

```bash
python run_benchmark.py --task 1 --model gpt-4.1
```

Key arguments:

| Argument | Description | Default |
|----------|-------------|---------|
| `--task` | Task number (1–4) | required |
| `--model` | Short model key or full OpenRouter string | required |
| `--col_name` | Column name in results CSV | last part of `--model` |
| `--reasoning` | Reasoning effort: `none` / `low` / `medium` / `high` | `none` |
| `--use_name` | Use compound names instead of SMILES (Task 4) | off |
| `--limit` | Process only first N rows (for testing) | — |
| `--overwrite` | Re-run already filled rows | off |
| `--data` | Override default data path (without extension) | — |

Supported model keys (mapped to OpenRouter strings):

```
claude-sonnet-4.6   claude-opus-4.6    claude-haiku-4.5
gemini-3-flash      gemini-2.5-flash   gemini-3.1-pro  gemini-3.1-pro(high)
gpt-4.1             gpt-5.2            gpt-5.2-name
glm-5               glm-4.7            kimi-k2.5
qwen3-235b          qwen3.5-397b-a17b  qwen-2.5-7b-instruct
llama-3.1-8b-instruct  llama-3.3-70b-instruct
grok-4.1-fast       deepseek-v3.2      gemma-3-27b-it
```

Results are saved incrementally to `results/task{N}_results.csv` after each row — safe to interrupt and resume.

## ML Baseline

A Leave-One-Compound-Out (LOCO) LightGBM baseline is provided in `ML_baseline/`. For each test solute, the model is trained on all remaining records from BigSolDB 2.0 (Tasks 1, 2, 4) or MixtureSolDB (Task 3), ensuring the test compound is never seen during training — mirroring the zero-shot condition of LLMs. Scripts and results for all four tasks are included.

## Reproducing benchmark data

Download the required source datasets and place them in the working directory:

| Dataset | Paper | Download |
|---------|-------|----------|
| `BigSolDBv2.0.csv` | [Nature Scientific Data](https://www.nature.com/articles/s41597-025-05559-8) | [Zenodo](https://doi.org/10.5281/zenodo.15094979) |
| `MixtureSolDB.csv` | [Nature Scientific Data](https://www.nature.com/articles/s41597-026-07047-z) | [Zenodo](https://doi.org/10.5281/zenodo.18660057) |

Then run:

```bash
python task_generation.py
```
