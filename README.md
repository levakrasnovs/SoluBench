# SoluBench

**SoluBench** is a benchmark for evaluating large language models on solubility prediction tasks of increasing complexity. It is built on top of [BigSolDB v2.0](https://www.nature.com/articles/s41597-025-05559-8) and [MixtureSolDB](https://doi.org/10.26434/chemrxiv-2025-m51v8) — two curated experimental solubility datasets.

## Tasks

| # | Name | Description | Input | Output |
|---|------|-------------|-------|--------|
| 1 | Pairwise solvent comparison | Given a compound (SMILES) and two solvents, select the one with higher solubility | SMILES + 2 solvents | A / B |
| 2 | Best solvent selection | Given a compound and a list of solvents, select the one with highest solubility | SMILES + N solvents | letter |
| 3 | Co-solvent effect prediction | Given a compound, a base solvent, and a co-solvent mixture, predict whether solubility increases or decreases | SMILES + mixture composition | A / B |
| 4 | Pairwise compound comparison | Given a solvent and two compounds (SMILES), select the one with higher solubility | Solvent + 2 SMILES | A / B |

## Repository structure

```
SoluBench/
├── data/                          # Benchmark data (CSV + JSONL)
│   ├── task1_pairwise_solvent_comparison.{csv,jsonl}
│   ├── task2_best_solvent_selection.{csv,jsonl}
│   ├── task3_cosolvent_effect_prediction.{csv,jsonl}

```

## Installation

```bash
git clone https://github.com/your-username/SoluBench.git
cd SoluBench
pip install -r requirements.txt
```

Set your [OpenRouter](https://openrouter.ai) API key:

```bash
echo "OPENROUTER_API_KEY=sk-or-..." > .env
```

---

## Running a model

```bash
python run_benchmark.py --task 1 --model gpt-4.1
```

Key arguments:

| Argument | Description | Default |
|----------|-------------|---------|
| `--task` | Task number (1–4) | required |
| `--model` | Short model key or full OpenRouter string | required |
| `--col_name` | Column name in results CSV | last part of `--model` |
| `--reasoning` | Reasoning effort: `none` / `low` / `medium` / `high` | `none` |
| `--use_name` | Use compound names instead of SMILES (Task 4) | off |
| `--limit` | Process only first N rows (for testing) | — |
| `--overwrite` | Re-run already filled rows | off |
| `--data` | Override default data path (without extension) | — |

Supported model keys (mapped to OpenRouter strings):

```
claude-sonnet-4.6   claude-opus-4.6    claude-haiku-4.5
gemini-3-flash      gemini-2.5-flash   gemini-3.1-pro  gemini-3.1-pro(high)
gpt-4.1             gpt-5.2            gpt-5.2-name
glm-5               glm-4.7            kimi-k2.5
qwen3-235b          qwen3.5-397b-a17b  qwen-2.5-7b-instruct
llama-3.1-8b-instruct  llama-3.3-70b-instruct
grok-4.1-fast       deepseek-v3.2      gemma-3-27b-it
```

Results are saved incrementally to `results/task{N}_results.csv` after each row — safe to interrupt and resume.

## Reproducing benchmark data

Download the required source datasets and place them in the working directory:

| Dataset | Paper | Download |
|---------|-------|----------|
| `BigSolDBv2.0.csv` | [Nature Scientific Data](https://www.nature.com/articles/s41597-025-05559-8) | [Zenodo](https://doi.org/10.5281/zenodo.15094979) |
| `MixtureSolDB.csv` | [ChemRxiv](https://doi.org/10.26434/chemrxiv-2025-m51v8) | [Zenodo](https://doi.org/10.5281/zenodo.18660057) |

Then run:

```bash
python task_generation.py
```
